# Roadmap — Anpexia

## Fase 1 — MVP (agora)

**Objetivo**: lançar uma plataforma funcional, bonita e que resolva problemas reais.

- [x] Definir arquitetura e stack
- [x] Documentar módulos e integrações
- [ ] Estrutura do backend (API, autenticação, multi-tenant)
- [ ] Schema do banco de dados (Prisma)
- [ ] Módulo: Dashboard
- [ ] Módulo: Clientes/Contatos
- [ ] Módulo: Estoque
- [ ] Módulo: Mensagens automáticas (Evolution API)
- [ ] Frontend: painel do cliente
- [ ] Admin: painel do Angel
- [ ] Landing page
- [ ] Deploy inicial (Railway + Vercel)
- [ ] Primeiro cliente

## Fase 2 — Consolidação (meses 2-4)

**Objetivo**: estabilizar a plataforma, conquistar os primeiros 10 clientes.

- [ ] Módulo: Agendamentos
- [ ] Módulo: Financeiro básico
- [ ] Relatórios em PDF
- [ ] Programa de indicação
- [ ] Melhorias baseadas no feedback dos primeiros clientes
- [ ] Integração com gateway de pagamento para cobranças automáticas

## Fase 3 — Crescimento (meses 4-8)

**Objetivo**: diferenciar a Anpexia no mercado, chegar a 30+ clientes.

- [ ] Relatórios avançados e customizáveis
- [ ] Integração com Google Ads
- [ ] Integração com Facebook/Instagram Ads
- [ ] Integração com Gmail
- [ ] App mobile (React Native ou PWA)
- [ ] Onboarding self-service para clientes simples

## Fase 4 — Escala (8-12+ meses)

**Objetivo**: escalar a operação, atender médias empresas.

- [ ] Integração com Canva
- [ ] Integração com marketplaces
- [ ] Multi-idioma
- [ ] API pública para integrações de terceiros
- [ ] Equipe de desenvolvimento
- [ ] Equipe de vendas/suporte

## Princípios do roadmap

1. **Lançar rápido, iterar sempre** — MVP primeiro, perfeição depois
2. **Feedback > planejamento** — os clientes reais vão direcionar as prioridades
3. **Cada fase paga a próxima** — a receita dos clientes financia o crescimento
4. **Não antecipar complexidade** — resolver problemas quando eles aparecerem
